## Что Это За Проект

KSD_AI — локальный Python-пайплайн для смыслового перевода Сири Гуру Грантх Сахиб на русский по методологии Karminder Singh Dhillon.

Основные слои:

1. `ksd_build_knowledge.py`
   Собирает `ksd_knowledge.db` из `.docx`, `.pdf` и `.txt` источников.

2. `ksd_ai_translator.py`
   Берёт ang JSON, группирует строки по шабдам, вытаскивает контекст из SQLite, отправляет промпты через Playwright в ChatGPT, парсит JSON-ответ и сохраняет результаты в `ksd_ang_json/` и `output/`.

3. `ksd_backup_db.py`
   Делает SQL-дамп и JSON-снапшот базы для версионирования и восстановления.

## Ключевые Артефакты

- `ksd_knowledge.db`: база знаний проекта
- `ksd_knowledge_dump.sql`: дамп SQLite для git
- `ksd_knowledge_snapshot.json`: краткий снапшот ключевых таблиц
- `ksd_ang_json/`: готовые KSD JSON-результаты
- `output/`: итоговые `.docx`
- `ksd_translator.progress`: прогресс длительного прогона

## Что Уже Выяснено

- `README.md` частично устарел: там упомянут "Claude API", но текущий `ksd_ai_translator.py` фактически работает через ChatGPT + Playwright.
- Переводчик — самая хрупкая часть системы, потому что зависит от UI `chatgpt.com`, браузерного профиля и строгого JSON-формата ответа.
- Источник истины для терминов и методологии здесь — база знаний и исходные документы, а не сам промпт.
- В `ksd_ai_translator.py` уже добавлены:
  - retrieval по шабду из `ksd_knowledge.db`;
  - fallback на `site_roman`, если `roman` пустой или битый;
  - `--print-prompt` для отладки prompt без запуска браузера;
  - проверка латиницы в русских полях с repair-проходом;
  - восстановление пропущенного `roman` при `--rebuild-docx`.
- В `ksd_build_knowledge.py` начата отдельная AST-индексация вместо грубого `extract_words_from_jbani`.
- AST-перевод Джап Джи нужно импортировать в БД как полезный, но не абсолютный источник:
  - его формулировки местами удачны и приоритетны как русский рендер;
  - но большой комментарий/словарь KSD должен иметь больший вес при корректировке спорных мест.
- Пользователь явно зафиксировал стиль перевода:
  - минималистично;
  - близко к тексту;
  - сохранять метафоры и логику строки;
  - не раздувать смысл;
  - не оставлять латиницу внутри русских полей;
  - транслитерация допустима отдельно, а не внутри русского перевода.

## Текущий Фокус

- Импортировать весь Джап Джи пользователя в внутреннюю БД.
- Затем использовать этот импорт как приоритетный reference при обновлении `ksd_ang_json` для ангов `1–8`.
- Для спорных мест (`Ind`, `Dhu`, и т.п.) не считать AST последней инстанцией; сверять с большим комментарием и словарём из Джап Бани.

## Риски

- В `ksd_ai_translator.py` путь `ANG_JSON_DIR` указывает на `../custom_khoj_sahib_singh/ang_json`, то есть на внешний каталог вне текущей папки проекта.
- В `ksd_build_knowledge.py` тоже есть внешние зависимости по путям (`ksd_reference_material`, `Nanak_Canvas`).
- Поэтому любые правки лучше начинать с проверки реальных путей и соответствия `README` фактическому коду.
