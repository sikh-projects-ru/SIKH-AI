# KSD_AI — AI Karminder Singh Dhillon

Система для создания аутентичного KSD-перевода СГГС на русский язык.

## Структура

```
KSD_AI/
├── ksd_build_knowledge.py   # Строит базу знаний из всех документов
├── ksd_ai_translator.py     # Основной переводчик (Claude API)
├── ksd_backup_db.py         # Бэкап/восстановление БД
├── ksd_knowledge.db         # SQLite база знаний
├── ksd_knowledge_dump.sql   # SQL-дамп для git
├── ksd_knowledge_snapshot.json  # JSON-снапшот ключевых таблиц
├── output/                  # Результаты перевода
│   ├── SGGS_KSD_Russian.docx   # Финальный docx
│   └── ksd_results.jsonl       # JSON лог всех переводов
└── logs/
```

## Запуск

```bash
# 1. Построить/обновить базу знаний
python3 ksd_build_knowledge.py

# 2. Перевести анг 1
ANTHROPIC_API_KEY=... python3 ksd_ai_translator.py --ang 1

# 3. Перевести Джап Джи (анги 1–8)
ANTHROPIC_API_KEY=... python3 ksd_ai_translator.py --ang 1-8

# 4. Продолжить с остановки
ANTHROPIC_API_KEY=... python3 ksd_ai_translator.py --resume

# 5. Бэкап БД
python3 ksd_backup_db.py

# 6. Восстановить БД из дампа
python3 ksd_backup_db.py --restore
```

## Формат вывода (docx)

Для каждой строки шабда:
- **Гурмукхи** (тёмно-красный)
- Романизация (серый курсив)
- KSD-перевод (тёмно-зелёный) — смысл выше художественности
- 〜 Художественный вариант (тёмно-синий курсив) — только если добавляет ценность
- Синий блок — "crossing over" момент (только там, где буквальное уводит в сторону)
- `[ Резюме шабда ]` — 1-2 предложения

## Принципы перевода

1. Порядок строк и шабдов — **не меняется никогда**
2. Смысл > художественность (художественный вариант — опциональная строка)
3. Confidence HIGH/MEDIUM/LOW на каждый вывод
4. Nanak Canvas как ограничение: смерть/реинкарнация/ад — не буквально
5. Гурбани объясняет Гурбани — без внешних философий
6. "Для Себя" — Гурбани про читателя, не про кого-то снаружи
