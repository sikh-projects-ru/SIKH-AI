# Sri Guru Granth Sahib RU — концепт Android-приложения

## Назначение

`Sri Guru Granth Sahib RU` — отдельное Android-приложение для чтения полного Sri Guru Granth Sahib на русском языке. Оно использует тот же визуальный стиль и reader-подход, что `Nitnem Authentic`, но строится вокруг полного корпуса SGGS: 1430 ангов, раги/разделы, авторы, шабды, строки и переводческие слои.

Рабочее имя:

```text
app name: Sri Guru Granth Sahib RU
package: org.ksd.sggsru
source data: KSD_AI/ksd_ang_json + sggs_meta
```

Приложение не генерирует перевод и не обращается к KSD-скриптам во время работы. Оно читает заранее подготовленный content pack.

## Почему не просто Nitnem

`Nitnem Authentic` — reader для малого набора: первые 13 ангов и Ананд Бани. Главный сценарий там простой: открыть ежедневное чтение и быстро перейти к известным произведениям.

`Sri Guru Granth Sahib RU` — библиотека полного SGGS. Главный сценарий другой:

- найти нужный анг;
- открыть раг или раздел;
- отфильтровать по автору;
- перейти к шабду;
- искать по тексту;
- быстро перелистывать анги через пагинатор;
- читать последовательно;
- сравнивать переводческие слои;
- позже искать по всему корпусу.

Поэтому код reader можно переиспользовать, но навигация, индекс и формат asset-пакета должны быть отдельными.

## Источник данных

К моменту создания концепта все 1430 ангов приведены к единому формату:

```text
ksd_ang_json/ksd_ang_0001.json ... ksd_ang_1430.json
```

Метаданные:

- `sggs_meta/raags.json` — раги и крупные разделы: id, порядок, названия, диапазон ангов, авторы.
- `sggs_meta/authors.json` — авторы: Гуру, бхагаты, бхатты и другие.
- `sggs_meta/shabad_index.json` — `shabad_id`, стартовый анг, raag_id, author_id, verse_count.

## Content Pack

Для SGGS не стоит собирать один огромный JSON как в Nitnem. Лучше сразу разнести пакет:

```text
sggs_app/content/
  sggs_manifest.json
  indexes/
    raags.json
    authors.json
    shabads.json
    works.json
  angs/
    ksd_ang_0001.json
    ...
    ksd_ang_1430.json
```

В Android assets:

```text
sggs_mobile/app/src/main/assets/sggs_ru/
  sggs_manifest.json
  indexes/
  angs/
  articles/
  images/
```

Преимущество: приложение загружает в память только manifest/indexes и текущий анг, а не весь корпус.

Первый релиз должен включать все 1430 ангов offline в APK. При этом данные всё равно лучше хранить по файлам, чтобы не держать весь корпус в памяти.

Нужна небольшая статья `about_sahib_singh_ru.md`: что русский перевод подготовлен по мотивам и линии толкования Prof. Sahib Singh, без заявления, что это официальный или буквальный перевод самого автора на русский.

## Главный Экран

Предложение:

```text
Top bar:
  Sri Guru Granth Sahib RU
  search
  settings

Tabs:
  Чтение
  Раги
  Авторы
  Разделы
  Анги
```

### Чтение / Анг

Основной экран чтения открывает выбранный анг. Не нужен отдельный стартовый экран "продолжить чтение" как обязательная точка входа: достаточно поиска, перехода по ангам и понятного текущего положения.

- выбранный анг;
- пагинатор ангов 1-1430 с отметкой текущего анга;
- предыдущий/следующий анг;
- список шабдов внутри анга;
- выделение Рахао;
- раскрытие/сворачивание строки;
- сохранение последней позиции.

### Раги

Список из `sggs_meta/raags.json`:

- название на русском;
- Gurmukhi;
- диапазон ангов;
- количество шабдов/строк;
- тип: preamble, raag, section, appendix;
- авторы внутри раздела.

При выборе рага:

- описание/диапазон;
- список ангов этого рага;
- список шабдов;
- фильтр по автору внутри рага.

### Авторы

Список из `sggs_meta/authors.json`:

- Гуру/Махалла;
- бхагаты;
- бхатты;
- другие авторы.

При выборе автора:

- краткая карточка;
- диапазон ангов;
- количество строк;
- список рагов/разделов, где есть автор;
- список шабдов автора.

### Разделы / Произведения

Это пользовательский слой поверх `raags.json`. Сюда можно включить:

- Джап;
- Со Дар;
- Со Пуркх;
- Сохила;
- Сири Раг;
- все остальные раги;
- Шлок Кабира;
- Шлок Фарида;
- Свайе;
- Мундхавани;
- Раг Мала.

Вопрос для обсуждения: называть вкладку `Разделы`, `Произведения`, `Содержание` или `СГГС`.

### Анги

- поле номера 1-1430;
- сетка/список ангов;
- текущий раг рядом с номером;
- продолжить с последнего открытого анга.

## Слои Чтения

Для первого релиза основной публичный слой — Prof. Sahib Singh:

- Gurmukhi;
- roman;
- Sahib Singh Punjabi;
- Sahib Singh Russian.

Пустые слои скрываются. `ksd_ru`, `ipotseluev_ru` и другие слои остаются в данных, но не должны засорять интерфейс, если для строки или анга они пустые.

Позже можно включить дополнительные переключатели:

- KSD русский основной;
- KSD художественный;
- KSD контекст;
- confidence/reason;
- I. Potseluev Russian.

## Поиск

MVP:

- поиск как основной пользовательский сценарий;
- поиск по текущему ангу;
- поиск по спискам рагов/авторов/ангов.

Следующий этап:

- полнотекстовый индекс по Gurmukhi;
- roman;
- `ksd_ru.main`;
- `sahib_singh_ru.main`;
- фильтр результатов по рагу, автору, диапазону ангов.

Для offline full-text лучше заранее генерировать индекс, а не сканировать 1430 JSON при каждом вводе.

## Архитектура

Предлагаемые папки:

```text
sggs_app/
  export_sggs_content.py
  content/
  schemas/

sggs_mobile/
  app/
    src/main/assets/sggs_ru/
    src/main/java/org/ksd/sggsru/
      MainActivity.kt
      data/
      domain/
      ui/
```

Первый MVP можно сделать копированием `nitnem_mobile`, но сразу переименовать:

- `rootProject.name = "SggsRu"`;
- `namespace = "org.ksd.sggsru"`;
- `applicationId = "org.ksd.sggsru"`;
- app label `Sri Guru Granth Sahib RU`;
- assets path `sggs_ru/...`;
- reader model под полный corpus.

Дизайн оставить тем же, что в Nitnem: палитра, карточки строк, логика слоёв и общий rhythm reading UI. Дальше желательно вынести общие reader-компоненты из Nitnem и SGGS в общий модуль, но это можно сделать после первого работающего прототипа.

## Индексация Баний и Произведений

Раги и крупные разделы уже индексируются через `sggs_meta/raags.json`. Но отдельные известные бани и композиции внутри SGGS, например:

- Ананд;
- Сукхмани;
- Аса ди Вар;
- Барах Маха;
- другие устойчивые произведения,

нужно индексировать отдельным пользовательским слоем `works.json`.

Если в текущих JSON нет достаточной разметки для таких произведений, это отдельная процедура: добавить или сгенерировать стабильные границы work/unit поверх `shabad_id`, `ang`, `verse_id`. Не нужно ломать основной `ksd_ang_json`; лучше держать это как индекс:

```json
{
  "id": "sukhmani",
  "title_ru": "Сукхмани",
  "title_gurmukhi": "ਸੁਖਮਨੀ",
  "ang_start": 262,
  "ang_end": 296,
  "shabad_ids": [],
  "unit_kind": "ashtpadi"
}
```

Для MVP достаточно заложить место под `works.json`; полную разметку известных баний можно сделать позже.

## MVP Реализации

1. Создать `sggs_app/export_sggs_content.py`.
2. Сгенерировать `sggs_manifest.json`, indexes и `angs/*.json`.
3. Добавить статью `about_sahib_singh_ru.md`.
4. Скопировать `nitnem_mobile` в `sggs_mobile`.
5. Переименовать package/app label.
6. Заменить загрузку `nitnem_ru_ksd_v1.json` на `sggs_ru/sggs_manifest.json` + загрузку текущего анга.
7. Оставить reader строк максимально близким к Nitnem.
8. Добавить вкладки `Раги`, `Авторы`, `Разделы`, `Анги`.
9. Добавить поиск и пагинатор ангов с отметкой текущего анга.
10. Добавить выбор анга через индексы.
11. Проверить сборку APK.
