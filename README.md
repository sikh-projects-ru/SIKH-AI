# Духовные переводы | Spiritual Translations

Проект по переводу сикхских духовных текстов на русский язык.  
Работа ведётся в диалоге с [Claude](https://claude.ai) (Anthropic) — все переводы, комментарии и решения принимаются в живом чате, а результаты сохраняются в кэш и собираются в DOCX.

## Подпроекты

### 1. KhojGurbani — перевод Джапу Джи Сахиб

**Источник:** [khojgurbani.org](https://khojgurbani.org) — уникальные переводы и комментарии Джапу Джи Сахиб на английском.  
**Выход:** `KhojGurbani_Russian.docx`  
**Скрипт:** `khojgurbani_translate.py`

Каждый шабад переводится в двух версиях:
- **KG (синий)** — перевод KhojGurbani (метафорический, Гурмат-ориентированный)
- **SS (зелёный)** — перевод проф. Сахиба Сингха (буквальный, с пунджаби)

После каждого шабада — комментарий KG с разбором терминов и толкованием.

**Прогресс:** АНГ 1–4 (пауди Шалок, 1–21), 199 строф, 22 комментария.

#### Workflow

```bash
# Посмотреть данные шабада из локального дампа (без сети)
python3 khojgurbani_translate.py show --ang 5 --shabad 23 --dump kg_raw_all.json

# Собрать DOCX для одного анга
python3 khojgurbani_translate.py build --ang 4 --shabad 19 --end-shabad 22 --output /tmp/ang4.docx

# Скачать сырые данные из API (для новых ангов)
python3 khojgurbani_translate.py dump --ang 5 --shabad 23 --end-shabad 27 --output kg_raw_ang5.json
```

**Кэш переводов:** `khojgurbani_cache.json` (426 ключей)  
Ключи: `kg_{id}` (KG перевод), `ss_{id}` (SS перевод), `commentary_{shabad_id}` (комментарий)

### 2. СГГС Дарпан (проф. Сахиб Сингх)

**Источник:** «Шри Гуру Грантх Дарпан» — классический построчный комментарий на пунджаби.  
**Выход:** `SGGS_Darpan_Russian.docx`  
**Скрипт:** `create_sggs_docx.py`

Цветовая схема блоков:
- `Bani2` → тёмно-красный (бани)
- `Arath` → синий (артх, смысл)
- `PadArath` → фиолетовый (словарь)
- `Note` → зелёный (примечания)
- `Bhav` → бирюзовый (общий смысл)

**Прогресс:** Введение + Мул-мантар + Шалок + пауди 1–16.

## Как устроена работа с Claude

Вся переводческая работа ведётся в режиме диалога:

1. Данные API загружаются локально (`dump` команда) — один раз, без трат токенов
2. Claude читает исходный текст через `show` и предлагает перевод в чате
3. Переводы записываются в JSON-кэш (`khojgurbani_cache.json`)
4. Готовый DOCX собирается скриптом (`build` команда)

История работы хранится в кэше и в git-коммитах — каждый коммит отражает реально завершённый блок перевода.

| Гурпрасад, Нирбхау, Нираваир | оставлять |

| Пауди, шабад, анг | транслитерировать |
